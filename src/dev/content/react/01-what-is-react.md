# 1장. React가 무엇인지 이해하기

React는 사용자 인터페이스를 만들기 위한 **JavaScript 라이브러리**이다. Angular처럼 라우팅·폼·HTTP까지 한 번에 정해 주는 **프레임워크**가 아니라, 화면을 **컴포넌트**라는 작은 단위로 나누어 조립하는 데 초점을 둔다. 라우팅(`react-router`)이나 상태 관리(예: Redux)는 필요할 때 별도 라이브러리로 덧붙인다.

## React가 해결하려는 문제

브라우저에서 화면을 바꿀 때마다 DOM을 직접 찾아서(`document.querySelector` 등) 속성과 자식 노드를 일일이 조작하면, 화면이 커질수록 코드가 얽히고 버그가 나기 쉽다. React는 **상태(state)** 가 바뀌면, 그 상태에 맞는 UI를 **선언적으로** 다시 그리도록 도와준다. 개발자는 “이 데이터가 있을 때 화면은 이렇게 보여야 한다”라고 적고, 실제 DOM에 무엇을 넣거나 빼야 할지는 React가 비교·갱신한다.

## DOM 직접 조작과의 차이

- **명령형**: “이 버튼을 누르면 저 요소의 `textContent`를 바꿔라.”
- **선언형(React)**: “`count` 값이 이만큼이면 화면에는 이 숫자를 보여줘라.”

## 기본 트리 구조: 화면은 한 그루 나무처럼 생긴다

웹 페이지의 HTML은 생각해 보면 **중첩된 트리**다. `<html>` 아래 `<body>`, 그 아래 `<div>`, 그 안에 `<header>`, `<main>`… 처럼 **부모 노드 안에 자식 노드**가 들어 간다. React도 같다. 다만 “태그 트리”가 아니라 **컴포넌트 트리**로 구성하는 것이 일반적이다.

### 루트와 마운트

실제 프로젝트에서는 보통 **하나의 DOM 요소**(예: `index.html`의 `<div id="root">`)를 정해 두고, 그 안에 React가 만든 UI 전체를 **한 번에 붙인다**(마운트). 진입 코드에서는 `createRoot(rootElement).render(<App />)`처럼 **최상위 컴포넌트**(여기서는 `App`)를 넘긴다. 사용자가 보는 화면 전체 트리는 이 `App`에서부터 아래로 뻗어 나간다고 보면 된다.

### 부모–자식 관계

- **부모 컴포넌트**가 JSX 안에서 **자식 컴포넌트**를 태그처럼 쓰면, 렌더링 결과에 그 자식이 **안쪽(하위)** 에 포함된다.
- 자식은 또 자신만의 자식을 가질 수 있다. 이렇게 내려가며 **깊이가 깊어질 수 있는 구조**가 곧 React 앱의 **UI 트리**다.

개념만 그림으로 보면 다음과 비슷하다.

```text
App (최상위, 페이지 골격)
├── Header
│   ├── Logo
│   └── Nav
├── Main
│   ├── PageTitle
│   └── ContentArea
│       └── Card   ← 필요하면 계속 아래로
└── Footer
```

실제 코드에서는 `App`의 `return` 안에 `<Header />`, `<Main />` … 를 나열하고, `Main` 안에서는 또 `<PageTitle />`, `<ContentArea />`처럼 쓴다. **“누가 누구를 화면에 포함하는가”** 가 트리의 **포함 관계**다.

### 한 컴포넌트가 반환하는 것도 작은 트리

함수 컴포넌트는 **하나의 루트** 아래에 여러 요소를 묶어 반환한다(형제 노드만 여러 개 던질 수 없고, `<></>` Fragment로 한 단계 감싸거나 상위 `div` 하나로 묶는다). 그 **반환값 JSX**도 작은 **서브트리**이고, 그 안에 또 다른 컴포넌트가 들어가면 트리가 이어진다.

### 트리 구조가 주는 이점

1. **역할 분담**: 화면을 처음부터 한 덩어리로 짜지 않고, 덩어리 단위로 파일·함수를 나누기 쉽다.
2. **재사용**: 같은 `Button`, `Card`를 트리 여러 위치에서 쓸 수 있다.
3. **사고하기 쉬움**: “지금 이 블록은 어떤 부모 아래에 붙어 있지?”만 따라가도 구조를 추적할 수 있다.

### 데이터는 아래로(일단 개념만)

UI 트리에서 **데이터를 자식에게 넘기는 통로**가 `props`이고, 이벤트·상태로 **화면을 바꾸는 이야기**는 뒤 장에서 이어진다. 여기서 기억할 것은 **트리는 “누가 누구를 감싸며 그린다”는 구조**이고, **값은 주로 부모에서 자식 방향으로 내려간다**는 점이다.

## 요약

- React는 UI를 **컴포넌트 트리**로 구성하는 쪽이 자연스럽다.
- **루트**에서 `App` 같은 최상위 컴포넌트를 마운트하고, 그 아래로 부모가 자식을 포함하며 트리가 깊어진다.
- 트리는 역할 분리·재사용·구조 파악에 도움이 된다.
- 데이터는 주로 **부모 → 자식**(`props`) 방향으로 내려가며, 상세는 다음 장들에서 다룬다.
- 이 연재에서는 JavaScript(및 JSX)만 사용해 예제를 맞춘다.
