# 16장. 작은 관리자 앱으로 마무리

이 연재는 **실제 `src/etc`를 그대로 베끼는 것**이 목표가 아니라, 같은 **구조와 흐름**을 **자바스크립트와 작은 예제**로 몸에 익히는 것이다. 마지막 장에서는 그동안의 조각을 **한 앱**으로 엮을 때의 **체크리스트**를 정리한다.

## 권장 구성(미니 프로젝트)

1. **라우팅** — 공개 영역과 `/admin` (또는 `/app`) 영역 분리, 로그인 제외 보호(9·11장).
2. **로그인** — 토큰 저장 정책을 단순히 잡고, API 모듈에 인증 헤더(10·11장).
3. **관리자 레이아웃** — 헤더·사이드·`Outlet`으로 자식 페이지만 전환(9장).
4. **아트워크 목록** — 검색·필터, 빈 상태(6·5장).
5. **등록·수정 폼** — 제어 컴포넌트, 검증, id 기반 수정(12장).
6. **이미지** — file 선택, 미리보기, 순서(옵션으로 간단 DnD 없이 위아래 버튼만)(14장).
7. **전역** — 로그인 사용자·목록 캐시 정도만 RTK로 시도해 볼 수 있음(13장).
8. **실시간 맛보기** — 고정 채팅방 하나, `WebSocket`으로 수신만(15장).

## 단계별로 늘리기

- **1단계**: 정적 목록 + 라우팅만.
- **2단계**: `fetch`로 목록, 로딩/에러.
- **3단계**: 로그인 쿠키/토큰 가정 후 보호 라우트.
- **4단계**: 폼 + `POST`/`PUT`.
- **5단계**: 이미지·Redux·WS 중 필요한 것만.

한 번에 다 넣으려 하면 막히기 쉽다.

## 품질에 손대는 순서

1. **에러 메시지** 사용자에게 닿게.
2. **중복 제출·빈 요청** 차단.
3. **로딩 스켈레톤** 또는 스피너 통일.
4. 접근성(포커스, 버튼 `type`).

## 이 프로젝트와의 대응

| 연재에서 연습한 것 | 실제 코드에서 볼 위치(예) |
|-------------------|---------------------------|
| 라우트·레이아웃 | `App.tsx`, `AdminLayout`, `PublicLayout` |
| API | `src/etc/api/*` |
| 폼·아트워크 | `ArtworkPost` 등 |
| 이미지·정렬 | `useImageUpload`, `SortableItem` |
| 채팅 | `useChatWebSocket`, `AdminChatPage` |

## 마무리

React는 **컴포넌트·상태·이벤트**에서 시작해 **모듈·훅·라우터·서버·전역 상태**로 확장된다. 각 장의 예제와 Live 데모를 손으로 다시 치고, 이 체크리스트대로 **작은 앱 하나**를 끝까지 올려 보면 실무 코드 읽기 속도가 확 달라진다.

## 요약

- 마지막은 **구조 복제**가 아니라 **흐름 이해 + 작게 재현**이다.
- 기능은 **단계적으로** 늘리고, 라우팅·인증·API·폼·미디어·실시간을 **순서 있게** 붙인다.
- `src/etc`는 **지도**로 삼아 같은 패턴을 찾아 읽는다.
